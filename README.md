本系列文档旨在描述需求文档，为需求提供标准化指导。


使用方式：请Fork出自己的版本，在修改后提交Pull Request到Master分支

### 标准格式

># 【模块名称】
>## 3.1 【功能场景：比如邀请式注册】
>#### 3.1.1.a 【功能点，a表示前台功能，b表示后台功能,前台功能和后台功能必须一一对应】
>##### 功能描述
>就是这么屌，你服不服

>##### *关联功能*
>主要描述前置功能，不用描述相关后台功能
>比如：要支持留言，必须要有用户登录，则用户登录为留言功能的“前置功能”
